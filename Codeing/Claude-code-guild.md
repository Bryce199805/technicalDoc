# Claude Code 用户手册 (操作指南)

Claude Code 是 Anthropic 推出的官方 CLI 工具，作为一个运行在终端中的交互式 AI 软件工程助手，它直接感知你的本地开发环境。本指南将详细说明如何使用它来执行日常的软件开发任务。

## 1. 安装与启动

**安装:**
目前建议通过 npm 全局安装：
```bash
npm install -g @anthropic-ai/claude-code
```

**启动:**
在你的项目目录（如 git 仓库）下，直接在终端中输入：
```bash
claude
```
启动后，你将进入一个交互式的命令行提示符，可以直接用自然语言向 Claude 下达指令。

---

## 2. 核心功能与操作方式

Claude Code 通过一系列**内置工具**来理解和修改你的代码。当你提出请求时，它会自动选择合适的工具执行操作。

### 2.1 文件操作 (无需手动指定，由指令触发)

*   **读取文件 (Read):**
    *   *你的指令示例:* "查看 `src/components/Button.jsx` 的内容并解释它是做什么的。"
    *   *背后行为:* Claude 会调用 Read 工具获取文件内容，然后分析并回复你。
*   **编辑文件 (Edit):**
    *   *你的指令示例:* "将 `utils/date.js` 中的 `formatDate` 函数改成驼峰命名法。"
    *   *背后行为:* Claude 会先读取文件，理解上下文，然后调用 Edit 工具**精确地**替换需要修改的字符串。它会倾向于修改现有文件而不是创建新文件。
*   **创建文件 (Write):**
    *   *你的指令示例:* "在 `tests` 目录下新建一个 `user.test.js`，包含基础的 Jest 测试脚手架。"
    *   *背后行为:* 只有在必须时，Claude 才会调用 Write 工具写入全新文件。

### 2.2 搜索与探索

*   **全局搜索文件 (Glob):**
    *   *你的指令示例:* "找到所有 `.test.js` 结尾的文件。"
    *   *背后行为:* Claude 使用 Glob 工具在文件树中查找匹配项，这比运行 `find` 命令更高效且更结构化。
*   **文件内容搜索 (Grep):**
    *   *你的指令示例:* "找到项目中所有用到 `fetchUserData` 的地方。"
    *   *背后行为:* 使用 Grep 工具进行正则搜索。
*   **深度探索代理 (Agent/Subagent):**
    *   *你的指令示例:* "深入研究这个项目的认证流程，并给我一个总结报告。"
    *   *背后行为:* 对于需要多次查询或广泛浏览的复杂任务，Claude 会启动子代理（Agent 工具），并行探索代码库，收集结果汇总后回复你。

### 2.3 执行命令与网络交互

*   **终端命令 (Bash):**
    *   *你的指令示例:* "运行 npm test 看一下哪些测试失败了。"
    *   *背后行为:* Claude 会调用 Bash 工具在你的终端中执行命令并读取输出（注意：涉及破坏性命令时会触发安全拦截）。
*   **网络获取 (WebFetch):**
    *   *你的指令示例:* "总结一下 `https://docs.anthropic.com/claude/docs` 这个页面的内容。"
    *   *背后行为:* 获取 URL 内容并转换为 Markdown 进行分析。

## 3. 快捷命令 (Slash Commands)

在 Claude 提示符下输入 `/` 开头的命令可以快速执行特定操作：

*   `/help` - 显示所有可用的快捷命令和使用说明。
*   `/fast` - 切换到“快速模式” (Fast mode)。在此模式下，它会使用相同的基础模型 (如 Opus)，但响应速度更快，适合简单、急需的任务。
*   `/commit` (如果支持此 Skill) - 自动分析当前的 git 修改，生成合适的 commit message 并执行提交。
*   `/simplify` (Skill) - 审查你最近修改的代码，检查复用性、质量和效率，并自动修复发现的问题。

## 4. 权限与安全机制

Claude Code 拥有直接操作你文件系统的能力，因此内置了安全机制：

*   **权限模式:** 当 Claude 尝试调用工具（如执行 bash 命令、修改文件）时，如果该操作超出了当前的自动允许范围，**终端会暂停并提示你批准 (Approve) 或拒绝 (Deny)**。
*   **不可逆操作确认:** 对于 `rm -rf`, 强制推送 `git push -f`, 删除数据库表等高风险操作，Claude 默认必须先向你说明意图，并等待你的明确同意。
*   **防误操作:** 如果你拒绝了它的某个工具调用，它不会死循环重试，而是会停下来分析原因或向你提问 (`AskUserQuestion`)。

## 5. 高级配置：自动记忆与 CLAUDE.md

### 5.1 自动记忆 (Auto Memory)

Claude Code 会跨会话持久化地记住关于你的项目的重要信息。
*   **存储位置:** `~/.claude/projects/` 或项目内的特定 `.claude/` 目录。
*   **它记住什么:** 重要的架构决策、文件路径、你纠正过它的错误模式、以及你明确要求它记住的工作流偏好（例如："以后永远使用 bun 而不是 npm"）。
*   **它不记住什么:** 临时的任务状态、未验证的推测。
*   **如何管理:** 你可以直接对它说 "忘掉关于 xxx 的设定"，它会自动去更新或删除记忆文件。

### 5.2 CLAUDE.md (项目级指令)

强烈建议在项目根目录创建一个 `CLAUDE.md` 文件。
*   **用途:** 为 Claude 提供该项目特定的、持久化的系统指令。
*   **内容示例:** 你可以在里面定义代码风格规范（如 "只使用 Tabs 不使用 Spaces"）、测试运行命令、甚至授权上下文（如 "允许在不确认的情况下自动运行本地单元测试"）。Claude 在启动时会自动加载并遵循此文件。

### 5.3 钩子 (Hooks)

你可以在设置中配置 Hooks，以便在特定事件（如每次代码提交前、或 Claude 调用特定工具后）触发自定义的 Shell 脚本，从而将 Claude 更好地集成到你现有的 CI/CD 或本地构建流程中。

## 6. 最佳实践指南

1.  **先读后改:** 在让 Claude 修改复杂文件前，先让它 "看一看" (`Read`)，理解上下文。
2.  **避免让它凭空创造:** 尽量让它修改现有文件，不要为了微小的改动让它新建一堆工具类或冗余文件。
3.  **不要过度工程:** 明确告诉它 "只修复这个 bug，不要重构周围的代码" 或者 "不需要添加额外的错误处理，保持最简"。
4.  **提供精确的错误信息:** 如果测试失败或编译报错，直接把终端里红色的错误栈粘贴给它，它能迅速定位。
5.  **代码位置引导:** 在对话中引用代码时，使用 `文件路径:行号` 的格式（如 `src/main.js:45`），可以帮助 Claude 更快定位。
