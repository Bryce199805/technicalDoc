# Markdown 基础语法

## 1. 标题 (Headings)

Markdown 支持 6 级标题，使用 `#` 号表示。

### 语法速查表

| 级别 | 语法 | 说明 |
|------|------|------|
| H1 | `# 一级标题` | 最高级标题，通常用于文档标题 |
| H2 | `## 二级标题` | 章节标题 |
| H3 | `### 三级标题` | 小节标题 |
| H4 | `#### 四级标题` | 段落标题 |
| H5 | `##### 五级标题` | 子段落标题 |
| H6 | `###### 六级标题` | 最小标题 |

### 替代语法

一级和二级标题可以使用替代语法：

```markdown
一级标题
========

二级标题
--------
```

### 示例

```markdown
# 这是一级标题

## 这是二级标题

### 这是三级标题
```

### 渲染效果

# 这是一级标题

## 这是二级标题

### 这是三级标题

## 2. 段落与换行

### 段落规则

| 规则 | 说明 |
|------|------|
| 段落分隔 | 使用一个或多个空行分隔段落 |
| 换行 | 行末加两个空格或使用 `<br>` |
| 连续文本 | 普通换行符会被当作空格处理 |

### 示例

```markdown
这是第一段落。

这是第二段落。

这是第三段落的
第一行。
这是第三段落的第二行。
```

## 3. 列表 (Lists)

### 3.1 无序列表

无序列表使用 `-`、`*` 或 `+` 作为标记。

| 语法 | 说明 |
|------|------|
| `- 项目` | 使用连字符（推荐） |
| `* 项目` | 使用星号 |
| `+ 项目` | 使用加号 |

```markdown
- 项目一
- 项目二
  - 子项目 2.1
  - 子项目 2.2
- 项目三
```

**渲染效果：**
- 项目一
- 项目二
  - 子项目 2.1
  - 子项目 2.2
- 项目三

### 3.2 有序列表

有序列表使用数字加点号。

```markdown
1. 第一项
2. 第二项
   1. 子项 2.1
   2. 子项 2.2
3. 第三项
```

**注意：** 列表序号不必按顺序，Markdown 会自动编号。

```markdown
1. 第一项
5. 第二项  <!-- 会渲染为 2. -->
3. 第三项  <!-- 会渲染为 3. -->
```

### 3.3 任务列表 (Task Lists)

GFM 扩展语法，创建待办事项列表。

```markdown
- [x] 已完成的任务
- [ ] 未完成的任务
- [ ] 另一个未完成任务
```

**渲染效果：**
- [x] 已完成的任务
- [ ] 未完成的任务
- [ ] 另一个未完成任务

### 3.4 定义列表

部分 Markdown 解析器支持定义列表。

```markdown
术语 1
: 定义 1

术语 2
: 定义 2a
: 定义 2b
```

### 3.5 列表嵌套规则

| 规则 | 说明 |
|------|------|
| 缩进 | 子列表需要 2-4 个空格缩进 |
| 混合 | 有序和无序列表可以互相嵌套 |
| 段落 | 列表项中可包含多个段落（用空行分隔并缩进） |

```markdown
1. 第一项段落一

   第一项段落二

2. 第二项
   - 子项 A
   - 子项 B
```

## 4. 链接 (Links)

### 4.1 链接语法表

| 类型 | 语法 | 说明 |
|------|------|------|
| 行内链接 | `[显示文本](URL)` | 最常用的链接方式 |
| 带标题链接 | `[显示文本](URL "标题")` | 鼠标悬停显示标题 |
| 引用式链接 | `[显示文本][引用ID]` | 文档末尾定义 URL |
| 自动链接 | `<URL>` | 自动转换为可点击链接 |
| 邮箱链接 | `<email@example.com>` | 自动转换为 mailto 链接 |

### 4.2 行内链接

```markdown
[GitHub](https://github.com)

[GitHub](https://github.com "访问 GitHub")
```

### 4.3 引用式链接

```markdown
[GitHub][1]
[Google][google]

[1]: https://github.com
[google]: https://google.com
```

### 4.4 相对路径链接

```markdown
[基础语法](./01-基础语法.md)
[上级目录](../README.md)
[项目根目录](/README.md)
```

## 5. 图片 (Images)

### 5.1 图片语法表

| 类型 | 语法 |
|------|------|
| 基本图片 | `![替代文本](图片URL)` |
| 带标题图片 | `![替代文本](图片URL "标题")` |
| 引用式图片 | `![替代文本][id]` |
| 指定尺寸 | `<img src="URL" width="200">` (HTML) |

### 5.2 示例

```markdown
![Markdown Logo](https://markdown-here.com/img/icon256.png)

![示例图片](./images/example.png "这是一个示例")

![引用图片][logo]

[logo]: ./images/logo.png "Logo"
```

### 5.3 图片链接

将图片包裹在链接语法中：

```markdown
[![GitHub](https://github.com/favicon.ico)](https://github.com)
```

## 6. 水平分隔线

使用三个或更多的 `-`、`*` 或 `_` 创建水平分隔线。

| 语法 | 说明 |
|------|------|
| `---` | 连字符（推荐） |
| `***` | 星号 |
| `___` | 下划线 |

```markdown
---

***

___
```

**注意：** 连字符前后需要空行，避免被解析为标题。

## 7. 引用块 (Blockquotes)

### 基本语法

使用 `>` 符号创建引用块。

```markdown
> 这是一个引用块。
> 可以跨越多行。
```

**渲染效果：**
> 这是一个引用块。
> 可以跨越多行。

### 嵌套引用

```markdown
> 第一层引用
>
> > 第二层引用
> >
> > > 第三层引用
```

### 引用中包含其他元素

```markdown
> ## 引用中的标题
>
> 1. 引用中的列表项一
> 2. 引用中的列表项二
>
> 引用中的**加粗**和*斜体*
```

## 8. 转义字符

使用反斜杠 `\` 转义 Markdown 特殊字符。

### 可转义字符表

| 字符 | 名称 | 字符 | 名称 |
|:----:|------|:----:|------|
| `\` | 反斜杠 | `` ` `` | 反引号 |
| `*` | 星号 | `_` | 下划线 |
| `{}` | 花括号 | `[]` | 方括号 |
| `()` | 圆括号 | `#` | 井号 |
| `+` | 加号 | `-` | 连字符 |
| `.` | 句点 | `!` | 感叹号 |
| `|` | 竖线 | | |

```markdown
\* 这不是斜体 \*
\# 这不是标题
```

## 9. HTML 元素

Markdown 支持内嵌 HTML 标签。

### 常用 HTML 标签

| 标签 | 用途 | 示例 |
|------|------|------|
| `<br>` | 换行 | `第一行<br>第二行` |
| `<sup>` | 上标 | `x<sup>2</sup>` → x² |
| `<sub>` | 下标 | `H<sub>2</sub>O` → H₂O |
| `<mark>` | 高亮 | `<mark>高亮文字</mark>` |
| `<kbd>` | 键盘按键 | `<kbd>Ctrl</kbd>` |
| `<details>` | 折叠内容 | 手风琴效果 |

### 折叠内容示例

```html
<details>
<summary>点击展开</summary>

这里是折叠的内容。

</details>
```

<details>
<summary>点击展开</summary>

这里是折叠的内容。

</details>

## 常见问题

### Q: 为什么我的换行没有生效？

A: Markdown 中普通换行符会被当作空格。解决方案：
- 行末添加两个空格
- 使用 `<br>` 标签
- 使用空行分隔段落

### Q: 如何在列表中使用代码块？

A: 代码块需要缩进 8 个空格或 2 个 tab。

```markdown
1. 列表项

        代码块（8空格缩进）
```

### Q: 为什么我的链接无法点击？

A: 检查以下几点：
- URL 是否完整（包含 `https://` 或 `http://`）
- 方括号和圆括号是否配对
- 是否有多余的空格
